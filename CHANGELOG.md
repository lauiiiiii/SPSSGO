# 更新日志

## [2026-05-23] 方差分析的精度跃迁

### 事后多重比较：修正了一项影响 SNK 与 Duncan 检验精度的关键缺陷

`_pair_p_value` 函数接收 `rank_distance` 参数以控制逐步检验口径，但调用方 `_comparison_rows` 未向该参数传入有效值。这导致 `max(None, 2)` 恒返回 2——SNK Q 检验与 Duncan 检验在任何均值排序间距下均退化至最宽松口径，逐步收紧与逐步放宽逻辑完全失效。

修正后，`_comparison_rows` 先按各组均值排序生成 `mean_rank` 映射，再对每一对比较计算真实的排序间距：`abs(mean_rank[a] - mean_rank[b]) + 1`。SNK 的 `k` 值从此按实际间距动态确定，Duncan 的多步校正指数同步恢复。

> 受影响的函数：`_pair_p_value`（第 128–160 行）、`_comparison_rows`（第 163–195 行）

### N 因素方差分析：自变量数量不再受限

传统统计软件将方差分析的自变量上限固定为二至三个，这一限制在 SPSSGO 被解除。新增的 `n_way_anova` 方法接受任意数量的分类自变量与连续协变量，自动生成主效应表及全部交互效应表。计算层由 R 引擎执行，输出精度与原生 R 一致。

分析参数文档已同步更新，前端配置面板通过 `factors` 多选槽位自动适配变量数量。

### Python–R 双引擎接口规范发布

新增接口对齐文档，完整描述 Python 侧传入参数与 R 侧接收参数的映射关系，覆盖变量名传递、矩阵转换与结果回传的全部环节。此后向 SPSSGO 添加 R 分析方法无需追溯源码推断参数格式。

### 自动化测试

`test_n_way_anova.py` 覆盖 N 因素方差分析的参数构造到结果输出全链路，核心路径回归验证纳入 CI。

## [2026-05-18] 方差分析重构与图表增强

### 新增分析方法

- **事后多重比较**：全面重构，支持 LSD、Bonferroni、Tukey、Scheffe、SNK、Duncan、Games-Howell 七种比较方法，新增字母标记法、效应量、P 值标注功能，附带均值对比图
- **双因素方差分析**：重构为统一的 factors 参数，新增协变量支持、交互效应开关、事后多重比较集成，附带均值对比图
- **三因素方差分析**：重构为统一的 factors 参数，新增协变量支持、二阶/三阶交互控制、效应量选项、事后多重比较集成

### 前端增强

- **图表系统**：指标对比图新增多系列折线图支持，展示多条数据序列对比，含图例和差异化配色
- **变量拖拽**：拖拽变量时显示拖拽数量预览标记，提升交互反馈
- **配置面板**：适配 factors（多选分类变量）槽位类型，支持协变量配置
- **方法元数据**：新增事后多重比较、双因素/三因素方差分析的前端槽位和选项配置
